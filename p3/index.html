The Physics of CSS

CSS is essentially mapping HTML elements to rectangular boxes that can be positioned on the x-, y- and z-axis.

There is a single place these HTML elements usually live in: the normal flow. Elements in the normal flow are born as inline, block elements, or not displayed. Not displayed elements are those like meta tags or style tags which aren’t visually rendered by the browser. However, HTML elements can also be born outside of the normal flow, and interact with the normal flow in their own way: like floats, relative elements, and fixed elements. Let’s take a look at inline and block elements.

Inline vs. Block

Every HTML element has three different states: Block, Inline or not displayed.

A block level element will naturally span the entire available width, without concern about how much horizontal space it actually needs. As a result of that, a block level element will automatically push following content to a new line, so by default, two block level elements can't stand next to each other - the first will push the second one to the next line (floating elements will allow this though, but more on that later).

However, an inline element does not break the current flow. An inline element only takes up the space it needs to render its content and after that, more inline elements can be displayed.

A good way to remember the difference is to think of an inline element as a line of text/sentence and a block level element as a paragraph - the sentence can stand together with other sentences to form an entire paragraph, but the paragraph breaks the natural flow to create space around it.

An example of inline elements include span tags (the generic inline element), links (anchors), images and form fields like input. Common block level elements include div tags (the generic block level element), header tags (h1-h6), paragraphs, lists and tables.

Another important fact is that width and height properties are ignored for inline elements. Instead, you can use the line-height property to make an inline element smaller or bigger, as the space between each line grows or shrinks.

You will also see margins and paddings acting differently than you might be used to, when applied to inline elements. In general, if you need to create space between or inside elements, you may want to use a block level element instead. If you're trying to use inline elements because you want two or more elements to stand next to each other, then a floating block level element might be more suitable.

Outline vs. Borders

While most HTML elements don't have a border by default, CSS does support the feature, with a range of border related properties.

Border width

The width can be either an absolute value, a relative value, or one of the pre-defined border width values: Thin, medium or thick. If you use the pre-defined values, it's up to the browser to interpret them, which basically gives you less control of how your work will look across all the different devices and browsers.

Border style

For style, you have a range of options. The most commonly used is the solid border, but there are many more to choose from: hidden, dotted, dashed, solid, double, groove, ridge, inset and outset.

Border radius

The ability to define border radius was added in CSS3, providing the possibility to make rounded corners. In terms of syntax, they’re four slots but the first one begins with the top-left and goes around clockwise. Like other properties, you can simplify the four slots to two slots or even one.

Difference Between Borders and Outlines

They look identical, however, they are a few key differences. You cannot apply a different outline width, style and color for the four sides of an element, like you can with the border - the values provided will be used for all four sides of the element.

In addition, the outline is not a part of the element's dimensions, like the border is, meaning that no matter how thick an outline you apply to the element, the dimensions of it won't change. This also means that the browser won't reserve the required space for your outline - you will need to make sure that it can fit it, without overlapping other elements.

Thus, it is not a feature that’s commonly used, however, it does have one unique feature that may come in handy.

Outline offset

A cool thing about outlines is that you can create a distance between it and the border, if you want to, by using the outline-offset property. Margins vs. Padding

Margins are used to ensure an element has a certain amount of space separating it from other elements. Think about it like an outer, invisible border. The margins of two elements can “overlap,” but in the case that it happens, the distance between them will be the greatest of each’s margin.

The default value for the margin properties is auto, which usually means no margin. Some elements, such as headers, body tags and paragraphs may have margins added by the browser. This, of course, can be overridden by setting margins to zero.

You can also have relative and negative margins. Using a combination of negative and positive values for the margin property is a common strategy to adjust the position and compensate for the padding in the parent element.

Padding is similar to margins, but on the inside of the element. In other words, padding is an inner, invisible border around your element.

Positioning

While HTML elements have properties, they also need to be positioned.

By default, elements are laid out with static positioning. This means elements are placed from top to bottom, with specified margins on the element and elements around it.

Relative positioning places an object relative to its default static position by using top, bottom, left, and right properties. Content around relatively positioned elements will not adjust to gaps or overlap created.

Fixed positioning positions an element relative to the browser window, and does not leave a gap in the normal flow of a page. Fixed elements are also placed using the top, bottom, left and right properties.

Absolute positioning places an element relative to the entire webpage, or within the confines of a non-static ancestor if one exists. Absolute positioning uses the same top, bottom, left and right properties.

Lastly, sticky positioning allows an object to transition from relative positioning to fixed positioning based on a user scrolling through a webpage. At least one offset must be specified for sticky positioning to work.

Syntax

Defining Properties

Syntax has been mentioned a few times, so let’s recap. When defining specific properties, for most of them, HTML provides up to four slots of information, starting with top and going around clockwise. For example, border-style: border-top-style, border-right-style, border-bottom-style and border-left-style. This pattern of top, right, bottom, and left is consistent with other properties, where each slot affects a side of the block element. However, much of the time, programmers use two slots or a single slot. In the case of a single slot, the browser interprets it such that the single slot is applied to all four sides.

Units

When defining properties, units also matter. There are three types: absolute, relative, and viewport.

The most common absolute unit are pixels, or px. However, a sometimes handy unit is em. Em refers to the size of the font of the most inner wrapping element, meaning it grows as the font grows.

The only relative unit is %. It takes a percentage of its parent element. However, it can be tricky sometimes, as the % may refer to different a different parent element than you have thought,

Lastly, viewport units. While relative units are responsive, viewport units are more responsive in the sense that their value changes every time the browser resizes. They’re two common ones: Viewport Height (vh, based on the height of the viewport) and Viewport Width (vw, based on the width of the viewport)

Selectors

You can select elements by id (unique identifiers) with a hash, class (identifiers for multiple elements) preceded by a period, or type of element.

#id

.class

type

Combining selectors without a space selects elements satisfying all combined selectors

Combining selectors with a space selects elements satisfying the later selector descended from the first selector

Separating selectors with commas selects elements satisfying any of the combined selectors.

There are more uncommon selector syntaxes, such as “element>element” (which denotes a parent and their child), “element+element” (which denotes the later element placed immediately after the former), and “element~element” (which denotes the later element preceded by the former).
